# i18n 使用说明（生产版）

## 目标
- 所有文本以 Key 为唯一来源，运行时通过字符串表取值并支持中英热切换。
- 资源（图片/音频）支持按语言后缀或回退加载。
- 新增语言以“新增表文件 + 配置字体/资源”完成，不需要改核心代码。

## 相关文档
- 数值配置（CSV 生效路径与工作流）：`Docs/GameConfig-Guide.md`
- v0.10 版本线回溯（本分支整理）：`Docs/Changelog-v0.10.md`

## 目录结构（运行时生效）
- `Assets/StreamingAssets/Localization/manifest.json`
- `Assets/StreamingAssets/Localization/core.zh-Hans.csv`
- `Assets/StreamingAssets/Localization/core.en.csv`

CSV 采用 `key,value` 两列，UTF-8 编码。示例：

```csv
key,value
ui.settings.fullscreen,全屏
```

## 运行时 API（代码里用）
- 读取文本：
  - `LocalizationManager.T("ui.settings.fullscreen")`
  - `LocalizationManager.TryGet("key", out var value)`
- 带参数文本（推荐用 `string.Format` 占位符）：
  - 字符串表：`ui.score,value: "Score: {0}"`
  - 代码：`LocalizationManager.Format("ui.score", score)`
- 语言切换：
  - `LocalizationManager.ChangeLanguage(LanguageId.En)`
  - `LocalizationManager.ChangeLanguage(LanguageId.ZhHans)`
- 预加载额外表（按需加载）：
  - `LocalizationManager.PreloadTable("ability")`
  - `LocalizationManager.PreloadTable("upgrade")`

## UI 组件（Prefab/Scene 里用）
- `LocalizedText`（UGUI Text）
- `LocalizedTMPText`（TextMeshPro）

把组件挂到对应 Text 上，填写 `Key`；运行时语言切换会自动刷新。调试时按 `F1` 会显示 `[Key]` 并高亮缺失翻译。

## 字体
- 可选：创建 `LocalizationFontCatalog`（Resources 根目录：`LocalizationFontCatalog`），配置不同语言对应字体。
- `FontManager` 会在语言变化时对已注册 Text/TMP_Text 应用字体；`LocalizedText/LocalizedTMPText` 会自动注册。

## 数字/日期/货币/排序
使用 `LocaleFormat`：
- `LocaleFormat.Number(123456)` -> 当前语言的千分位格式
- `LocaleFormat.DateTime(System.DateTime.Now, "yyyy-MM-dd")`
- `LocaleFormat.Currency(12.34m, "$")`
- `LocaleFormat.Sort(list)`

## 图片/音频本地化
- `LocalizedSpriteFromAtlas`：根据语言后缀优先加载 atlas（例如 `icon_en`、`icon_zh_Hans`），找不到则回退 `icon`。
- `LocalizedAudioClip`：根据语言后缀优先加载音频（例如 `voice_en`），找不到回退原 key。

后缀解析由 `LocalizationManager.AssetResolver` 决定，默认是“`{base}_{lang}` / `{base}_{langSafe}` / `{base}`”三段回退。

## 翻译工作流（导出/导入）
打开菜单：`VampireSurvivorLike/Localization Workbench`
- Export：把 runtime 表合并导出到 `Docs/LocalizationExport.csv`，并生成 `Docs/LocalizationExport.meta.json`（用于冲突检测）
- Import：把合并 CSV 写回 runtime 表文件；如检测到 runtime 表在导出后被改动，会在 `Assets/StreamingAssets/Localization/import.conflicts.txt` 记录冲突表

导出的合并 CSV 默认表头为：`table,key,zh-Hans,en,...`，Import 会按 table 列写回对应的 `{table}.{lang}.csv`。

## 新增语言（不改核心代码）
1. 增加表文件：例如 `core.ja.csv`、`ability.ja.csv` 等（同 `key,value` 格式）
2. 更新 `manifest.json` 的 `Languages` 列表
3. （可选）在 `LocalizationFontCatalog` 增加该语言字体
4. 运行时通过 `LocalizationManager.ChangeLanguage(new LanguageId("ja"))` 或 UI 切换生效
