# AI 功能创意方案 —— 类幸存者游戏

> 本文档列举 10 种可集成到本项目的 AI 类型，涵盖从经典游戏 AI 到大语言模型的广泛方向。

---

## 1. 动态难度调整 AI（Dynamic Difficulty Adjustment）

**核心思路**：实时监控玩家表现指标，自动调整游戏参数，让游戏始终保持在"心流区间"。

**监控指标**：
- 每分钟受伤次数、当前 HP 占比
- 每分钟击杀数、屏幕上存活敌人数量
- 升级选择速度（快速点击 = 无聊/过于简单）

**调整手段**：
- 动态修改 `EnemyWaveConfig` 中的生成间隔与敌人 HP/速度缩放
- 控制精英/Boss 的出现时机提前或延后
- 增减掉落率（玩家濒死时提升回血概率）

**与现有代码的结合点**：
- 写一个 `DDASystem : AbstractSystem`，读取 `Global` 中的 HP、击杀数等 `BindableProperty`
- 修改 `EnemyGenerator` 中的 `SpawnChannel` 参数而不是改配置文件

## 2. 群体智能敌人 AI（Swarm Intelligence / Boids）

**核心思路**：敌人不再各自独立直冲玩家，而是通过三条规则形成自组织群体行为：
- **分离**（Separation）：避免与同类重叠
- **对齐**（Alignment）：向同伴平均方向靠拢
- **聚合**（Cohesion）：向群体中心靠近

**效果**：
- 小型敌人形成"潮水"效果，包抄路线更真实
- 群体会绕开障碍物而不是卡墙
- 精英怪可作为"领袖节点"吸引周边小怪聚拢

**与现有代码的结合点**：
- 利用已有的 `EnemySpatialIndex` / `SpatialHashGrid` 查询周边邻居，成本极低
- 在 `Enemy.cs` 的移动逻辑中叠加 Boids 力向量

**性能注意**：每帧只对视野内的敌人执行，并利用 `EnemySimulationManager` 的批处理机制。

## 3. Boss 战术行为树 AI（Boss Behavior Tree）

**核心思路**：用行为树（Behavior Tree）替代当前 Boss 的简单技能轮转，让 Boss 能"读懂"战场并做出战术决策。

**决策示例**：
```
Root (Selector)
├── 玩家距离 < 2m → 近战冲撞
├── 玩家 HP < 30% → 减速区域压制（防止逃跑）
├── 小怪数量 < 5 → 召唤技能
└── 默认 → 投射物攻击
```

**进阶行为**：
- Boss 会"记忆"玩家的移动惯性，预判落点投弹
- 血量阶段切换时改变整套行为策略（狂暴/召唤/远程）
- 多 Boss 同场时互相配合（一个控场，一个输出）

**与现有代码的结合点**：
- 扩展 `IBossSkill` 接口，在 `BossSkillBase` 上层添加行为树调度层
- 现有 7 种技能（`AreaAttackSkill`、`DashSkill`、`SpinAttackSkill` 等）直接复用为行为树叶节点

## 4. 智能掉落推荐 AI（Smart Loot Weighting）

**核心思路**：分析玩家当前 Build，对掉落池进行概率权重调整，让游戏"引导"玩家走向有趣的协同路线。

**决策逻辑**：
- 玩家已有 SimpleSword → 提升 SimpleCritical（配对被动）出现权重
- 玩家满级某武器 → 降低该武器的再次出现概率
- 玩家 HP 长期偏低 → 提升回血道具掉落率

**反作弊机制**：
- 不完全由 AI 控制，保留随机性下限，避免游戏变成"必然路线"
- 玩家可在设置中关闭"智能推荐"，改用纯随机

**与现有代码的结合点**：
- 在 `ExpUpgradeSystem.Roll()` 中注入权重计算层
- 读取 `Global` 中已解锁的武器列表与 `ExpUpgradeSystem.Pairs` 配对关系

## 5. AI 自动游玩代理（Autopilot Agent）

**核心思路**：一个能自主游玩游戏的 AI 代理，用于自动化测试、平衡验证与演示模式。

**两种实现路线**：

**方案 A：规则驱动**（简单，适合立即实现）
- 移动：始终向敌人密度最低的方向移动（利用 `SpatialHashGrid` 查密度）
- 升级选择：按预设优先级表选择（攻击 > 暴击 > 速度）

**方案 B：强化学习**（复杂，长期目标）
- 状态空间：HP、等级、周边敌人密度、已有武器
- 动作空间：8 方向移动 + 升级选择
- 奖励函数：存活时间 + 击杀数 - 受伤次数
- 使用 ML-Agents 包训练

**用途**：
- 每次发版前自动跑 100 局，统计平均存活时间验证平衡性
- 作为"演示模式"在标题界面自动游玩

## 6. 程序化波次生成 AI（Procedural Wave Generator）

**核心思路**：不依赖手写 CSV，而是由 AI 根据目标难度曲线自动生成波次配置。

**生成约束**：
- 总体"威胁值"曲线：前期缓坡，中期陡增，后期Boss高峰
- 敌人种类多样性：同一时段不超过 X 种类型
- 保证 Boss 前有足够的小怪"热身"

**算法选项**：
- **约束满足（CSP）**：定义约束条件，随机搜索满足条件的配置
- **遗传算法**：以"玩家平均存活时间接近目标值"为适应度函数进化配置
- **噪声曲线**：用 Perlin Noise 生成威胁值曲线，再映射到敌人参数

**输出格式**：直接生成兼容现有 `EnemyWaveConfig.csv` 格式的配置，无需改动 `EnemyGenerator`。

## 7. 玩家画像与个性化挑战 AI（Player Profiling）

**核心思路**：跨局追踪玩家行为模式，为不同玩家生成个性化挑战。

**画像维度**：
- **战斗风格**：激进（主动追怪）/ 保守（绕圈走位）
- **Build 偏好**：近战型 / 远程型 / 范围型
- **决策速度**：快速直觉型 / 慢速分析型

**个性化应用**：
- 激进型玩家 → 生成更多需要走位规避的投射物敌人
- 近战偏好玩家 → Boss 多用远程技能逼迫走位调整
- 在成就系统中解锁"个性化称号"反映玩家风格

**存储方式**：扩展现有 `SaveSystem`（PlayerPrefs），存储局数统计数据。

## 8. 大语言模型叙事 AI（LLM Narrative Commentary）

**核心思路**：接入 Claude API（或本地小模型），在关键游戏事件时生成动态台词与叙事文本。

**触发时机与示例输出**：

| 事件 | AI 生成台词示例 |
|------|----------------|
| 玩家首次击杀 Boss | *"第一个倒下的强者……它们的同伴会记住你。"* |
| 玩家连续 5 次选同一武器 | *"你对那把剑似乎有着异常的执念。"* |
| 玩家 HP 跌至 10% 后逆转 | *"死亡只是一个开始——你用行动证明了这一点。"* |
| 游戏通关 | *"黎明终于到来。但黑暗……从未真正离去。"* |

**技术实现**：
- 游戏事件 → 构造 Prompt（包含当前局面状态）→ 调用 Claude API → 显示在屏幕角落
- 使用 `FloatingTextController` 或新增专用叙事文本 UI
- 离线缓存：预生成 200 条台词存为 JSON，无网络时随机抽取

**集成参考**：见 `Docs/AgentMemory-Guide.md`，项目已有 `AgentMemory` 系统可存储叙事上下文。

## 9. 过程生成地图 AI（Procedural Environment Generation）

**核心思路**：每局生成不同的地形障碍布局，打破"空旷平地"的单一感，迫使玩家调整走位策略。

**生成内容**：
- **地形障碍**：石柱、墙体、水坑（减速区）的随机摆放
- **功能区域**：安全屋（周期性出现，短暂无敌）、陷阱区（踩踏受伤）
- **动态事件**：AI 在特定时间点生成"环境变化"（涨潮、起火区域扩展）

**生成算法**：
- 用 **Wave Function Collapse** 生成符合逻辑的 Tile 布局
- 或用 **Cellular Automata** 生成有机形态的障碍簇

**平衡约束**：
- 保证玩家初始位置周围有足够走位空间
- 障碍不能完全封堵逃跑路线（连通性检验）
- 与现有 `RepeatTileController` 的无限背景系统兼容

## 10. 协同进化 AI（Co-evolutionary Arms Race）

**核心思路**：玩家与敌人相互进化——玩家变强，敌人进化出克制当前 Build 的新形态；形成永不结束的"军备竞赛"。

**进化机制**：
- 系统记录"哪种敌人最快被玩家消灭"
- 被快速消灭的敌人类型获得进化点数
- 积累足够点数后，该类敌人在下一局获得新特性：
  - 近战武器克制型 → 敌人获得"弹射伤害返还"护盾
  - 远程武器克制型 → 敌人获得"冲刺穿越投射物"能力
  - 范围武器克制型 → 敌人获得"分散站位"群体意识

**跨局持久化**：
- 进化数据存储在 `SaveSystem` 中，每局结束后更新
- 提供"重置进化"选项，让玩家从头开始

**与成就系统结合**：
- 触发特定进化组合解锁隐藏成就
- "见证进化"系列成就记录玩家对抗进化敌人的过程
