## 现状调研结论
- 业务代码大量硬编码中文（例如 [UIGamePanel.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/UI/UIGamePanel.cs#L49-L102)、[ExpUpgradeSystem.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/System/ExpUpgrade/ExpUpgradeSystem.cs#L80-L177)、[CoinUpgradeSystem.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/System/CoinUpgrade/CoinUpgradeSystem.cs#L20-L66)）。
- 工程内虽有 QFramework LocaleKit，但依赖的 `LanguageDefineConfig` Resources 资产当前未落地，且该方案是“每个 Text 挂多语言文本列表”，不适合你要求的表驱动/翻译工作流/冲突处理。
- 运行时资源加载基于 ResLoader + AssetBundle（WebGL 还有预加载逻辑：[WebGLPreloader.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/Global/WebGLPreloader.cs#L13-L64)），配置表来自 StreamingAssets/Config 的 CSV（例如技能名/描述在 [AbilityConfig.csv](file:///d:/unity/Vampire%20Survivor-like/Assets/StreamingAssets/Config/AbilityConfig.csv#L1-L6) 里是中文）。

## 总体方案（核心不改，扩展靠插件）
- 新增一套“Key/StringTable”式 i18n 子系统（与 QFramework 解耦、可插拔）：
  - 语言检测/切换/存储/回退：统一由 `LocalizationManager` 管理，`PlayerPrefs` 持久化；默认语言 + 回退链（如 `en -> zh-Hans`）。
  - 资源来源可插拔：字符串表 provider（StreamingAssets CSV/JSON、TextAsset、远端等）；图片/音频 resolver（按语言后缀/目录/manifest 解析）。
  - 运行时热切换：语言变化事件驱动 UI 组件与业务文本刷新，无需重启。

## 1) 多语言管理系统
- 新增 `Assets/Scripts/Localization/`：
  - `LanguageId`（如 `zh-Hans` / `en`，后续可扩展 `ja`/`ko`/`fr`）。
  - `LocalizationManager`：
    - `CurrentLanguage`（BindableProperty）+ `OnLanguageChanged`。
    - `Initialize()`：系统语言检测（`Application.systemLanguage`）、读取存档、加载基础表、设置 culture。
    - `ChangeLanguage(lang)`：按需加载该语言表并触发刷新；失败则按回退链取值。
  - `LocalizationSettings`（ScriptableObject）：默认语言、回退链、语言->CultureInfo 映射、语言->字体映射、表清单。

## 2) 本地化资源文件结构（覆盖所有文本）
- 新建运行时目录：`Assets/StreamingAssets/Localization/`
  - `manifest.json`：声明支持语言、表名、版本/hash、可选依赖（用于按需加载与冲突检测）。
  - 表文件：按模块拆表（提升加载与维护效率），例如：
    - `core.zh-Hans.csv` / `core.en.csv`（UI 通用/提示）
    - `upgrade.zh-Hans.csv` / `upgrade.en.csv`（升级相关）
    - `ability.zh-Hans.csv` / `ability.en.csv`（技能/道具名与描述）
- Key 命名规范：`ui.game.enemy_count`、`upgrade.exp.simple_sword.lv1`、`ability.simple_sword.name` 等。

## 3) 运行时语言切换
- 新增 UI 控件（优先放在设置面板 [UIGameSettingsPanel.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/UI/UIGameSettingsPanel.cs#L23-L129) 的 ScrollView 里）：语言下拉/Toggle。
- 切换时：保存 -> `LocalizationManager.ChangeLanguage()` -> 全局刷新。

## 4) 字体管理系统（中英+简繁扩展）
- 新增 `FontManager`：监听语言变化，给所有注册的 Text/TMP_Text 统一切换字体。
- 支持：
  - UGUI `Text.font` 切换
  - TextMeshPro（项目已引入 `com.unity.textmeshpro`）：TMP_FontAsset + fallback 链（简繁/日韩可扩展）。
- 产出一个 `LocaleFontConfig` 资产：语言 -> 字体（并支持默认/回退）。

## 5) 特殊本地化（数值/日期/货币/排序）
- 提供 `LocaleFormat` 工具：
  - `FormatNumber(value, style)`、`FormatDateTime(dt, pattern)`、`FormatCurrency(value, currencyCode)`
  - `Compare(a,b)`/`Sort(list)`：基于语言对应 CultureInfo 的 CompareInfo。
- 默认策略：不强行全局修改线程 Culture；业务通过工具方法输出（更可控）。

## 6) 图片/音频等资源本地化
- 新增 `ILocalizedAssetResolver` + 默认实现：
  - 规则 1：优先加载 `{baseKey}_{lang}`，否则回退 `{baseKey}`。
  - 规则 2：可选目录规则 `L10n/{lang}/{baseKey}`（便于美术打包）。
- 提供组件：`LocalizedSprite`、`LocalizedAudioClip`（可选 `LocalizedSpriteAtlas` 用于 atlas）。
- WebGL 预加载：[WebGLPreloader.cs](file:///d:/unity/Vampire%20Survivor-like/Assets/Scripts/Global/WebGLPreloader.cs#L34-L48) 按当前语言预加载对应字体/atlas；如启用多语言资源热切换，则按需再加载并缓存。

## 7) 翻译工作流（导出/导入/版本控制/冲突）
- 新增 Editor 工具 `LocalizationWorkbench`（`Assets/Editor/Localization/`）：
  - 扫描 Key 使用：
    - 代码中 `L.T("...")` / `L.Format("...")`
    - 场景/Prefab 上的 `LocalizedText` 组件
  - 导出：生成 `LocalizationExport.csv`（key、zh-Hans、en、context、table）。
  - 导入：合并回表文件并重新生成运行时 CSV/JSON；对 manifest/hash 不匹配的条目输出 `*.conflicts.csv`（保留双方内容）。

## 8) 自动化测试（文本/布局/编码）
- 使用 Unity Test Framework（项目已包含 `com.unity.test-framework`）：
  - EditMode：CSV/JSON 解析、回退链、缺失 key 检测、重复 key 检测、格式化与 culture 行为。
  - PlayMode：
    - 运行时切换语言，断言 Text/TMP_Text 内容即时更新
    - 缺失翻译时显示占位/高亮策略生效
    - 简单布局适配：对关键 UI Prefab 计算 preferredSize 与容器宽高阈值（避免英文变长溢出）。

## 9) 开发工具（运行时看 TextID/定位未翻译）
- Runtime Debug：
  - `LocalizationDebugOverlay`：可快捷键切换显示 key；缺失翻译文本用颜色/前缀标记；支持一键复制 key。
  - `LocalizationDiagnostics`：统计缺失翻译、未使用 key、重复 key（可在开发版输出）。

## 10) 插件化扩展（新增语言无需改核心代码）
- 核心仅依赖接口：
  - `IStringTableProvider`（加载字符串表）
  - `ILocalizedAssetResolver`（解析本地化资源）
  - `ILocaleFormatter`（格式化/排序策略）
- 新增语言流程：添加语言条目到 `LocalizationSettings` + 增加表文件/字体映射/（可选）资源目录，即可生效。

## 11) 性能与内存
- 字符串表按需加载：仅加载 `core`，其余表在首次请求或显式 `PreloadTables()` 时加载。
- 缓存策略：当前语言全缓存 + 最近一次语言 LRU（可配）；切换语言时可选择保留/释放。
- Asset 本地化：对 Sprite/Audio 使用缓存与引用计数，语言切换时只替换需要替换的组件。

## 12) 文档与示例

  - 如何定义 key、如何在代码/UI 中使用、如何做带参数文本
  - 如何新增语言、字体与本地化资源
  - 翻译导出/导入与冲突处理
- 提供示例：一个小场景/Prefab 展示 Text、TMP、图片、音频在运行时切换语言。
