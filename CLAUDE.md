# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Unity 2D (URP 2D) Vampire Survivors-like game prototype. Built with **Unity 2022.3.62f2c1 LTS**, **URP 14.x (2D)**, and **QFramework** (BindableProperty / UIKit / ResKit / AudioKit).

## Running the Project

Open in Unity Hub, then:
- `Assets/Scenes/GameStart.unity` — main menu entry point
- `Assets/Scenes/Game.unity` — battle scene

For builds: add both scenes to Build Settings > Scenes In Build. The custom `com.bshsf.appautobuilder` package supports multi-platform automated builds.

Performance/stress test scenes: `TestMaxEnemyCount.unity`, `TestMaxPowerUpCount.unity`.

## Architecture

### Core Layers

| Layer | Path | Purpose |
|-------|------|---------|
| **Global** | `Assets/Scripts/Global/Global.cs` | 100+ `BindableProperty<T>` reactive state properties; auto-initialized via `[RuntimeInitializeOnLoadMethod]`; auto-saved to `PlayerPrefs` |
| **Config** | `Assets/Scripts/Config/` | Static numeric constants (`Config.cs`) + CSV loaders for waves/abilities |
| **System** | `Assets/Scripts/System/` | Business systems inheriting `AbstractSystem`: `ExpUpgradeSystem`, `CoinUpgradeSystem`, `AchievementSystem`, `SaveSystem`, `DamageSystem`, performance subsystems |
| **Game** | `Assets/Scripts/Game/` | Battle logic: Player, Enemy, Ability (weapons), PowerUp (loot), EnemyGenerator, FX |
| **UI** | `Assets/Scripts/UI/` | `UIPanel` subclasses opened/closed via `UIKit.OpenPanel<T>()` / `ClosePanel<T>()` |
| **Localization** | `Assets/Scripts/Localization/` | Custom CSV-based i18n system (not Unity Localization package) |

All scripts use `namespace VampireSurvivorLike`.

### ViewController + Designer Pattern

Business logic lives in `XXX.cs`; Unity component references are auto-generated by QFramework into `XXX.Designer.cs`. Both use `partial class`. **Never manually edit `.Designer.cs` files.**

```csharp
// Player.cs — hand-written logic
public partial class Player : ViewController { ... }

// Player.Designer.cs — auto-generated, do not touch
public partial class Player {
    public UnityEngine.Animator Sprite;
    public CircleCollider2D HurtBox;
}
```

### Reactive Data Binding

```csharp
Global.HP.RegisterWithInitValue(hp => {
    // runs immediately and on every HP change
}).UnRegisterWhenGameObjectDestroyed(gameObject);
```

Always unregister callbacks — use `UnRegisterWhenGameObjectDestroyed` or manual `UnRegister` on destroy.

## Key Systems

### Weapon/Ability System

Weapons live in `Assets/Scripts/Game/Ability/`. Each extends `ViewController`. Object-pooled projectiles are suffixed `Pooled*Projectile.cs`. Add new weapons by following `SimpleSword.cs` as a reference.

### Wave/Enemy Generation

`EnemyGenerator.cs` drives spawning via game clock (not wave-clear). Supports two config modes:
- **CSV**: `Assets/StreamingAssets/Config/EnemyWaveConfig.csv`
- **ScriptableObject**: `LevelConfig` → `EnemyWaveGroup` → `EnemyWave` list

### Upgrade Systems

- **In-combat** (`ExpUpgradeSystem`): `Global.Level` change → `Roll()` → pause + show 3 options. Weapon pairing defined in `ExpUpgradeSystem.Pairs`. Synthesis triggered by `TreasureChest` pickup.
- **Permanent** (`CoinUpgradeSystem`): coin-purchased, persisted to `PlayerPrefs`.
- **Achievements** (`AchievementSystem`): registered in `OnInit()` via `Add(new AchievementItem(...))`.

### Collision & Damage

`HitHurtBox` merges HitBox + HurtBox into one component, using `Owner` tag to determine affiliation. All damage flows through `DamageSystem.CalculateDamage()` for crit/multiplier application.

```csharp
HurtBox.OnTriggerEnter2DEvent(col => {
    var hitHurtBox = col.GetComponent<HitHurtBox>();
    if (hitHurtBox?.Owner.CompareTag("Enemy")) { ... }
});
```

### Loot (PowerUp)

All pickups inherit `PowerUp` abstract class, implement `Execute()`, and use `FlyingToPlayer` for the magnet animation. Add new loot in `Assets/Scripts/Game/PowerUp/`.

### Performance Systems (`Assets/Scripts/System/Performance/`)

- `ObjectPoolSystem` — projectile/effect pooling
- `EnemySpatialIndex` / `SpatialHashGrid` — spatial proximity queries
- `PcInstancedEnemyRenderer` — GPU instancing for large enemy counts
- `MobileAdaptivePerformanceController` — quality scaling on mobile
- `PowerUpMergeSystem` — batches nearby loot pickups

### Audio

```csharp
AudioKit.PlaySound(Sfx.WALK, loop: true);  // enum-based, looping
AudioKit.PlaySound("Hit");                  // one-shot by name
```

Sound effect throttling is handled by `SfxThrottle.cs`.

## Quick Extension Guide

- **New weapon**: create `ViewController` subclass in `Assets/Scripts/Game/Ability/`, model after `SimpleSword.cs`
- **New loot**: create `PowerUp` subclass in `Assets/Scripts/Game/PowerUp/`, implement `Execute()`
- **New upgrade option**: call `Add(new ExpUpgradeItem(...))` in `ExpUpgradeSystem.ResetData()`
- **New achievement**: call `Add(new AchievementItem(...))` in `AchievementSystem.OnInit()`
- **New UI panel**: inherit `UIPanel`, open with `UIKit.OpenPanel<T>()`

## Pausing

Use `Time.timeScale = 0` / `1` to pause/resume (used during level-up and chest panels).
